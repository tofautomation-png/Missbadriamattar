Targetime • Website Development and Design

Project Document — Miss Badria Mattar

Scope: Framer website frontend + WordPress LMS + secure paywalls + Stripe payments. Designed for smooth student onboarding, crisp UX, and reliable subscription/checkout flows.

Timeline: 45 - 60 working days

01 · Project Overview

We will build a conversion‑focused website in Framer and a scalable WordPress LMS (e.g., LearnDash or TutorLMS). Access to premium lessons is controlled by Stripe (subscriptions or one‑time), with automatic paywalls and student enrollment sync. UX prioritizes clarity, speed, and mobile usability.

02 · Deliverables

Framer Website

  • Responsive pages: Home, About, Courses, Pricing, Testimonials, Contact.
  • Hero with value prop + clear enrollment path.
  • Trust elements: logos, testimonials, FAQs, compliance notes.
  • SEO basics: meta titles/descriptions, open graph.

WordPress LMS

  • Courses → modules → lessons → quizzes/certificates.
  • Student dashboard & progress tracking.
  • Content protection by membership/plan.
  • Email notifications (enrollment, password reset, receipts).

Stripe + Paywalls

  • Stripe account setup & product/price creation.
  • Checkout (subscription or one‑time) + webhooks.
  • Automatic access grant/revoke on payment status.
  • Invoices/receipts + dunning (failed payment) handling.
  • Payment methods: Apple Pay, Google Pay, Link, debit/credit cards.

Testing & QA

  • Cross‑browser/device checks (Chrome, Safari, iOS, Android).
  • Payment flow tests (test & live modes).
  • Enrollment and lesson access verification.
  • Performance passes (LCP/CLS), accessibility basics.

03 · Technical Stack

  • Frontend: Framer (responsive components, forms, SEO).
  • LMS: WordPress + (LearnDash or TutorLMS) + membership plugin if needed.
  • Payments: Stripe (Checkout/Customer Portal) + WP Stripe plugin + webhooks.
  • Hosting: Secure WP hosting (SSL, backups, caching/CDN).
  • Automation (optional): Email sequences for new students.
